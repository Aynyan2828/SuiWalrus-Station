## 1. プロジェクト概要 (Project Overview)

### 1.1 背景と目的
Suiブロックチェーンと分散型ストレージWalrusは強力な機能を備えていますが、CLI操作が中心であるため、一般ユーザーや初心者にとって参入障壁が高いという課題がありました。
**SuiWalrus Station** は、これらの強力な機能を安全なローカルデスクトップ環境でラップし、直感的なGUIとAIインテント・インターフェースを提供することで、「誰もが自律的なDeFi運用とWeb3ストレージ管理を享受できる環境」を構築することを目的としています。

### 1.2 ターゲットユーザー
- **DeFi投資家**: 自動積立（DCA）やステーキングを安全・簡単に運用したいユーザー。
- **Walrus開発者/クリエイター**: Blob管理やWebサイトデプロイを視覚的に管理したいユーザー。
- **セキュリティ重視のユーザー**: 秘密鍵をクラウドに預けず、ローカルで完全管理したいユーザー。

### 1.3 コア・コンセプト
- **Local-First Security**: 署名は常に端末内で行い、鍵情報は外部送信しない。
- **Intent-based AI**: 複雑なコマンドではなく「意図」をチャットで伝えるだけで実行。
- **Transparency**: 手数料計算や実行ロジックを100%可視化。

## 2. システム構成 (System Architecture)

### 2.1 テクニカルスタック
- **Frontend**: React 18, Vite, TypeScript, Vanilla CSS (Premium Glassmorphism).
- **Desktop Framework**: Tauri v2 (Rust-based secure bridge).
- **Blockchain Connectivity**: 
  - **Sui SDK**: `@mysten/sui` によるオンチェーン直接操作。
  - **CLI Wrapper**: インストール済みの `sui` / `walrus` CLIプロセスをスポーンして実行。
- **DeFi Protocols**: Cetus Aggregator SDK (Swap), Navi SDK (Lend/Supply).
- **AI / LLM**: Ollama (Local) または OpenAI 互換 API による自然言語パース。
- **Oracle**: Pyth Network (Hermes) によるリアルタイム価格取得。

## 3. 画面一覧 & 実装済み機能 (Implemented Features)

### 3.1 Dashboard (ダッシュボード)
アプリの起動画面であり、全システムのヘルスチェックと資産概要を提供。
- **総資産サマリー**: アクティブウォレットのSUI残高、および主要トークン残高のリアルタイム表示。
- **CLI接続ステータス**: Sui, Walrus, Site Builder 各CLIの検知状態とバージョンの可視化。
- **環境検知**: 設定ディレクトリ（`.sui`, `.walrus`）および `keystore` パスの自動スキャンと表示。
- **AI状態表示**: AI Guardモード（判断＋確認ダイアログ）のON/OFF状態を表示。
- **Portfolio Growth Chart**: 24時間ごとの資産スナップショットに基づく、過去30日間の資産推移SVGグラフの表示。
- **Walrus Agent Insight**: AIが資産状況と行動履歴を分析し、パーソナライズされた助言（博多弁）を表示するインサイト・ウィジェット。
- **グローバルログ**: 背後で動く全CLIコマンドとシステムメッセージをリアルタイム表示（最小化・クリア機能付）。

### 3.2 Wallet & Sui SDK (ウォレット・高度なSui操作)
SDK経由の直接操作とCLIコマンドを統合した管理画面。
- **ウォレット管理**:
  - `sui client addresses` との同期。
  - **新規アドレス生成**: Ed25519方式による新規キープリセット生成。
  - **秘密鍵インポート**: 33/32バイト秘密鍵（Sui形式, Hex, Base64）の取り込み。
  - **エクスポート**: 選択したアドレスの秘密鍵を安全に表示（確認ダイアログ付）。
- **資産操作 (Coin Manage)**:
  - 断片化したコインオブジェクトの統合（Merge）。
  - 指定額へのコイン分割（Split）。
- **NFT / デジタル資産**: IPFSおよびWalrusにホストされたメディアのプレビューと一覧表示、送金機能、および Tradeport SDK 連携による直接出品機能。
- **Staking & Yield**:
  - バリデーター一覧と最新APY（年利）の取得・表示。
  - 指定バリデーターへのステーキングおよびアンステーク（報酬回収含む）。
  - **Auto-compound (自動収穫)**: Navi Protocol の報酬を一括回収し、自動で再供給（Supply）する複利最大化エンジン。
- **PTBインスペクター**: 複雑な一括処理（Move Call, Transfer等）の結果を事前にログ確認して実行。

### 3.3 AI Agent (自律型エージェント)
自然言語による戦略構築と自動実行エンジン。
- **インテント解析**: 「毎日 9:00 に 10 USDC を SUI に替える」といった自然文から `AgentRule` を生成。
- **ルール管理 (CRUD)**:
  - **作成**: チャット入力またはマニュアルフォーム入力。
  - **編集**: 実行金額、頻度、時間、価格しきい値、スリッページ設定の変更。
  - **削除**: 保存済みルールの完全抹消。
  - **トグル**: ルールの有効/無効化の状態制御。
- **実行戦略 (Strategy Types)**:
  - `daily_dca_swap`: 毎日指定時間に自動スワップ（積立）。
  - `deposit_to_navi`: 余剰資金を指定プロトコルへ自動預け入れ。
  - `condition_threshold`: Pyth価格を監視し、指定価格以下/以上で発動（価格トリガー）。
- **タスクキュー & 承認**:
  - `manual_only`: 常にユーザーの最終確認後に実行。
  - `first_5_manual`: 最初の5回で学習・信頼し、以降は自動化。
  - `auto`: 条件合致で即座にバックグラウンド実行。

### 3.4 Walrus Tools (分散型ストレージ管理)
Walrusネットワークへのデータ配置とWebサイト管理。
- **Blob Explorer**: アップロード済みデータのBlob ID、サイズ、期間、ステータスの一覧表示。
- **Site Builder 連携**: `site-config.yaml` をスキャンし、デプロイ履歴をリスト化。
- **SuiNS マッピング**: Site Object と SuiNS ドメインの紐付け状況をオンチェーンから解決して表示。
- **ポータルアクセス**: `walrus.site` 等のポータルURLを自動生成。

### 3.5 Settings (システム設定)
- **CLIパス設定**: 手動または自動検出による各種CLIバイナリのパス管理。
- **AIプロバイダー設定**: モデル選択（Qwen, Llama3等）、APIキー、ベースURL設定。
- **AI接続接続テスト**: 最大30分のロングタイムアウト設定により、重いローカルLLMの初回起動を確実に待機。
- **手数料情報**: 現在の手数料率 (3%) と収集先アドレスの透明な表示。

## 4. 手数料仕様 (Fee Specifications)

### 4.1 徴収ロジック (Fee Engine)
- **適用対象**: AIエージェントによる「自動実行」および「スケジュール実行」。
- **料率**: 実行元本の **3%** (300 bps)。
- **除外条件**: 
  - テストネット/デブネットでの実行 (0%)。
  - ユーザーによる手動操作・手動CLI実行。

### 4.2 アトミック性の確保 (Atomic PTB)
手数料徴収は、メインの取引（SwapやDeposit）と同一の **Programmable Transaction Block (PTB)** 内に組み込まれます。
これにより、**「メインの取引が失敗したが手数料だけ徴収される」という事象は物理的に発生しません。** 取引全体が成功した時のみ、一括して決済されます。

## 5. セキュリティ仕様 (Security)

- **完全ローカル管理**: 秘密鍵情報は `~/.sui/sui.keystore` に保存され、アプリが独自にシークレットをクラウドへ保存することはありません。
- **サンドボックス実行**: 全てのオンチェーン操作は、公式のSui CLIプロセスを通じて安全に署名・ブロードキャストされます。
- **AIガードレール**: AIが生成した実行プランが不審な場合、ユーザーに警告ダイアログを表示する機能を搭載。
